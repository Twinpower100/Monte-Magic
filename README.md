# Monte Magic

Современный мультиязычный лендинг для заказа экскурсий, трансферов и индивидуальных туров по Черногории. Проект представляет собой веб-приложение с интегрированной Headless CMS для управления всем контентом на сайте.

## Особенности проекта

- **Мультиязычность**: Встроенная поддержка 4 языков (RU, EN, DE, ME) с автоматическим роутингом через Middleware.
- **Управление контентом (CMS)**: Удобная админ-панель на базе Payload CMS (полностью русифицирована) для управления экскурсиями, расписанием, командой и настройками сайта.
- **Бронирование**: Форма заявок с клиентской и серверной валидацией, rate limiting и защитой от спама (honeypot). Заявки сохраняются в базу данных.
- **Уведомления**: Опциональная отправка уведомлений о новых заявках на Email (SMTP) и в Telegram.
- **Развертывание**: Полная контейнеризация проекта с помощью Docker и Docker Compose.

## Технологический стек

- **Frontend**: Next.js 15, React 19, TypeScript, Tailwind CSS
- **Backend & CMS**: Payload CMS 3
- **База данных**: PostgreSQL 18
- **Инфраструктура**: Docker, Docker Compose

## Структура контента

Контент разделен на логические коллекции и глобальные настройки:

**Коллекции (Collections)**:
- `tours` (Экскурсии)
- `schedule-items` (Расписание выездов)
- `team-members` (Команда)
- `inquiry-requests` (Заявки пользователей)
- `media` (Загруженные файлы и изображения)
- `users` (Администраторы системы)

**Глобальные настройки (Globals)**:
- `home-page` (Контент главной страницы)
- `transfers-settings` (Трансферы)
- `partners-settings` (Партнерам)
- `contact-settings` (Контакты)
- `social-links` (Социальные сети)

## Запуск проекта локально

### Требования
- Установленный [Docker](https://www.docker.com/) и Docker Compose.
- [Node.js](https://nodejs.org/) (версия 20+), если планируете запускать приложение вне Docker.

### Шаг 1: Настройка окружения
Скопируйте пример конфигурационного файла и задайте свои значения (пароли, ключи, токены):
```bash
cp .env.example .env
```

### Шаг 2: Запуск через Docker (Рекомендуется)
Поднимите сразу и базу данных, и само приложение одной командой:
```bash
docker compose up -d --build
```
Приложение будет доступно по адресу: [http://localhost:3006](http://localhost:3006)
Админ-панель: [http://localhost:3006/admin](http://localhost:3006/admin)

*(Альтернативно: можно поднять в Docker только PostgreSQL `docker compose up -d postgres`, а приложение запустить локально через `npm run dev`)*.

### Шаг 3: Наполнение демо-данными (Seeding)
Чтобы не начинать с пустой базы, вы можете загрузить начальный контент (экскурсии, расписание, главную страницу и настройки):
```bash
npm run seed
```
*Если в `.env` заданы `ADMIN_EMAIL` и `ADMIN_PASSWORD`, скрипт автоматически создаст первого пользователя-администратора.*

## Полезные команды

- `npm run dev` — Запуск локального сервера для разработки.
- `npm run build` — Сборка production-версии проекта.
- `npm run start` — Запуск собранной production-версии.
- `npm run seed` — Заполнение базы данных стартовым контентом.
- `npm run generate:types` — Генерация TypeScript-типов на основе схемы Payload CMS.
