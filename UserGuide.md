# Инструкция по работе с админкой Goose Garden

Админка открывается по адресу:

**[https://goose-garden.ru/admin](https://goose-garden.ru/admin)**

Если домен будет другим, путь обычно такой же: **`/admin`**.

**SEO** (поисковая оптимизация, *Search Engine Optimization*) — далее по документу для краткости используется только аббревиатура **SEO**.

---

## 1. Вход в админку

1. Открой страницу `/admin`.
2. Введи электронную почту и пароль администратора.
3. Нажми кнопку входа.
4. После входа откроется главная страница панели управления.

Если это первый вход и учётная запись администратора ещё не создана, сначала создаётся первый пользователь с правами администратора.

---

## 2. Что есть в админке

В русифицированной админке основные разделы:


| Раздел в меню         | Назначение                                         |
| --------------------- | -------------------------------------------------- |
| **Настройки сайта**   | Название, тэглайн, логотип, SEO по умолчанию       |
| **Контакты**          | Почта, Telegram, WhatsApp, шаблоны сообщений       |
| **Социальные сети**   | Ссылки на Facebook, Instagram, Telegram            |
| **Главная страница**  | Все блоки главной (hero, о проекте, услуги и т.д.) |
| **Проекты портфолио** | Кейсы для портфолио                                |
| **Заявки**            | Обращения с формы на сайте                         |
| **Медиа**             | Библиотека файлов                                  |
| **Администраторы**    | Учётные записи с доступом в админку                |


Логика работы:

- тексты и общие настройки меняются в соответствующих глобальных разделах;
- фотографии и видео сначала загружаются в раздел **Медиа**;
- затем эти файлы выбираются в нужных местах сайта;
- проекты портфолио создаются отдельно;
- заявки с сайта приходят в раздел **Заявки**.

---

## 3. Главное правило по фотографиям и видео

В админке нет отдельного поля, где задаётся размер изображения или видео. Админка принимает файл как есть. Внешний вид на сайте зависит от того, **какой файл ты загрузил**.

Соблюдай рекомендуемые пропорции и размеры.

### Рекомендуемые размеры файлов


| Где используется       | Пропорция     | Рекомендуемый размер |
| ---------------------- | ------------- | -------------------- |
| Главный экран (hero)   | 16:9 или шире | от 2400×1350 px      |
| Блок «О проекте»       | 4:5           | от 1600×2000 px      |
| Обложка проекта        | 3:2 или 16:10 | от 2200×1400 px      |
| Фотографии галереи     | 4:3           | от 1600×1200 px      |
| Иконка услуги          | 1:1           | от 256×256 px        |
| Видео проекта          | 16:9          | 1920×1080 px         |
| Постер для видео       | 16:9          | 1920×1080 px         |
| Логотип горизонтальный | примерно 3:1  | 1200×400 px          |
| Логотип квадратный     | 1:1           | 800×800 px           |


### Технические правила

- для фотографий лучше использовать **JPG** или **WEBP**;
- для иконок лучше **PNG** или **SVG**;
- для видео лучше **MP4**;
- не загружай слишком маленькие изображения — они будут размытыми;
- не используй вертикальное видео для блока проекта: на сайте видео показывается в широком формате;
- у каждого изображения в **Медиа** обязательно заполняй поле **Alt-текст** (для русской и английской версии сайта — на соответствующих языковых вкладках).

---

## 4. Раздел «Медиа»

Библиотека всех файлов сайта: фотографии, видео, иконки, логотипы, постеры.

### Как загрузить файл

1. Открой раздел **Медиа**.
2. Создай новую запись (**Create new** / «Создать»).
3. Загрузи файл.
4. Заполни поле **Alt-текст** (на вкладках **Русский** и **English**, если сайт двуязычный).
5. Сохрани.

### К чему это приведёт на сайте

Сам по себе файл на странице не появится. Он станет доступен для выбора в других разделах:

- на главной странице;
- в проектах портфолио;
- в логотипе;
- в иконках услуг;
- в видео и постерах.

---

## 5. Раздел «Настройки сайта»

Основные параметры бренда и SEO по умолчанию.

### Что заполнять

- **Название сайта** — бренд, например «Goose Garden».
- **Тэглайн** — короткая фраза под названием.
- **Логотип** — файл из **Медиа**.
- **SEO по умолчанию** — группа полей:
  - **Title** — заголовок по умолчанию для поисковиков;
  - **Description** — описание по умолчанию.

### Как это влияет на сайт

- название и тэглайн используются в интерфейсе;
- логотип показывается в шапке и связанных блоках;
- **Title** и **Description** подставляются, когда у страницы нет своих SEO-полей.

### Логотип

- длинный горизонтальный знак — загружай горизонтальную версию;
- монограмма или знак — можно квадратную версию.

---

## 6. Раздел «Контакты»

Способы связи и тексты для быстрых сообщений (почта, мессенджеры).

### Что заполнять

- **Email**
- **Telegram username**
- **WhatsApp номер**
- **Шаблоны сообщений**:
  - тема email (**Тема email**);
  - текст письма (**Текст email**);
  - текст для Telegram;
  - текст для WhatsApp.

### Как это влияет на сайт

- в контактном блоке отображаются контакты;
- ссылки на почту, Telegram и WhatsApp формируются автоматически;
- при переходе по ссылкам пользователю может подставляться готовый текст из шаблонов.

### Как заполнять правильно

- почту указывай рабочую;
- имя в Telegram — с `@` или без, но лучше в одном формате везде;
- номер WhatsApp — в международном формате.

## 7. Раздел «Социальные сети»

Ссылки на страницы в соцсетях: **Facebook URL**, **Instagram URL**, **Telegram URL**.

На сайте появятся соответствующие ссылки; по клику пользователь переходит на указанную страницу.

---

## 8. Раздел «Главная страница»

Основной раздел содержимого главной. Внутри — сворачиваемые блоки. В интерфейсе Payload часть блоков подписана на английском (**Hero**, **CTA**) — ниже указано соответствие.

### 8.1. Блок «Hero» (главный экран)

Первый экран сайта.

**Поля:** заголовок hero, подзаголовок hero, текст кнопки hero, изображение hero.

**Изображение:** пропорция 16:9 или шире, от 2400×1350 px, выразительное широкое фото.

На сайте картинка растягивается на первый экран. Маленькое или узкое фото ухудшит вид блока.

### 8.2. Блок «О проекте»

Философия, подход, бренд.

**Поля:** заголовок блока, текст блока, изображение блока.

**Изображение:** 4:5, от 1600×2000 px.

На сайте — крупный блок с текстом и вертикально ориентированным фото.

### 8.3. Блок «Услуги»

**Поля:** заголовок секции, вступительный текст, список услуг.

Для каждой услуги: **Название**, **Описание**, **Иконка** (из **Медиа**).

**Иконка:** 1:1, от 256×256 px, простая читаемая графика.

На сайте — карточки услуг; иконка в круглом блоке.

### 8.4. Блок «Портфолио»

Только заголовок и вступительный текст секции портфолио на главной.

Сами проекты создаются в разделе **Проекты портфолио**.

### 8.5. Блок «Процесс работы»

**Поля:** заголовок секции, вступительный текст, шаги процесса.

Для каждого шага: **Номер шага**, **Название шага**, **Описание шага**.

### 8.6. Блок «CTA» (призыв к действию)

В панели блок называется **CTA**.

**Поля:** заголовок CTA, текст CTA, текст кнопки CTA.

Продающий блок перед формой заявки.

### 8.7. Блок «Контактная секция»

Текст над формой связи: заголовок секции, текст секции.

---

## 9. Раздел «Проекты портфолио»

### 9.1. Как создать проект

1. Открой **Проекты портфолио**.
2. Создай новую запись.
3. Заполни основную информацию.
4. При необходимости добавь галерею и видео.
5. Заполни SEO.
6. Сохрани.

### 9.2. Основная информация

**Поля:** название проекта, **Slug**, формат проекта, **Опубликован**, **Показывать на главной**, **Порядок вывода**, краткое описание, полное описание, **Обложка проекта**.

#### Что такое slug

Slug — часть URL проекта. Например, `modern-garden`. Должен быть уникальным, коротким, без пробелов, обычно латиницей.

#### Форматы проекта

- **Фотогалерея** — в основном фото.
- **Видео-кейс** — акцент на видео.
- **Смешанный кейс** — и фото, и видео.

**Обложка:** 3:2 или 16:10, от 2200×1400 px.

**Флаги:** «Опубликован» — виден на сайте; «Показывать на главной» — в блоке портфолио на главной; «Порядок вывода» — сортировка среди проектов.

### 9.3. Блок «Галерея»

**Поля:** изображения галереи (массив кадров, в каждом — загрузка из **Медиа**), **Автопрокрутка**, **Скорость автопрокрутки** (если включена автопрокрутка), **Ручная навигация**.

**Фото:** 4:3, от 1600×1200 px, по возможности единый стиль.

### 9.4. Блок «Видео»

**Источник видео:**

- без видео;
- загруженный файл;
- внешний URL.

Далее — **Видеофайл** или **Внешняя ссылка на видео**, **Постер видео**.

**Настройки воспроизведения:** автовоспроизведение, без звука, зациклить, показывать контролы, **Inline на мобильных**.

**Видео:** MP4, 16:9, 1920×1080, по возможности сжать по весу.  
**Постер:** 16:9, 1920×1080.

Видео показывается в широком блоке; вертикальные ролики не подходят.

### 9.5. Блок «SEO» (поисковая оптимизация, *Search Engine Optimization*)

**Поля:** **SEO title**, **SEO description** (локализуемые).

Для проекта в поиске используются эти значения вместо общих из «Настройки сайта».

## 10. Раздел «Заявки»

Заявки с формы на сайте. Внутренний раздел для администратора.

**Поля записи (как в базе):**

- имя;
- телефон;
- email;
- предпочтительный способ связи;
- локация проекта;
- тип услуги;
- краткое описание запроса;
- дополнительный комментарий;
- согласие на обработку персональных данных.

На публичном сайте этот раздел не отображается — только просмотр и работа с обращениями.

---

## 11. Раздел «Администраторы»

Учётные записи с доступом в админку (**коллекция users** в Payload).

Ты можешь создавать пользователей, менять email и пароль, при необходимости отключать доступ.

Доступ выдавай только тем, кто реально ведёт сайт.

---

## 12. Русская и английская версии

Сайт двуязычный. Многие текстовые поля локализованы.

- Вкладка **Русский** — текст для `/ru/…`
- Вкладка **English** — текст для `/en/…`

Если английскую вкладку не заполнишь, на английской версии тексты могут быть пустыми или подтянется fallback — лучше заполняй обе, где нужен английский.

---

## 13. Что проверять после изменений

После сохранения открой сайт и проверь:

- отображается ли нужный текст;
- загрузилось ли изображение;
- не обрезан ли важный объект на фото;
- открывается ли видео;
- проект в нужном месте и с нужным порядком;
- ссылки на почту, Telegram и WhatsApp;
- тестовая заявка попала в **Заявки**.

---

## 14. Краткая схема работы

1. Загрузи файлы в **Медиа** (с **Alt-текстом**).
2. Выбери их в глобалах и проектах.
3. Тексты главной — **Главная страница**.
4. Контакты и шаблоны — **Контакты**.
5. Соцсети — **Социальные сети**.
6. Общий бренд и SEO по умолчанию — **Настройки сайта**.
7. Проекты — **Проекты портфолио**.
8. Обращения — **Заявки**.
